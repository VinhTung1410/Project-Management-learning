# Agile & Scrum Fundamentals – Simplified Study Notes

## 1. Product Backlog
The Product Backlog is a living, evolving list of everything needed to build a product.

- Owned by the Product Owner
- Prioritized by business value
- Continuously refined

### Backlog Item Includes
- Description
- Business value
- Effort estimate

## 2. User Stories
User stories describe features from the user's perspective.

**Template**
As a <user>, I want <action> so that <value>.

**Example**
As an avid reader, I want to read reviews so that I can choose better books.

## 3. I.N.V.E.S.T. Criteria
- Independent
- Negotiable
- Valuable
- Estimable
- Small
- Testable

## 4. Epics
Epics are large user stories that group related stories.

## 5. Backlog Refinement
Ongoing process to clarify, estimate, and prioritize backlog items.

## 6. Estimation Techniques
- Planning Poker
- Dot Voting
- Bucket System
- Affinity Mapping

## 8. Sprints
Sprints are fixed-length iterations (1–4 weeks) that deliver value.

## 9. Sprint Events
- Sprint Planning
- Daily Scrum
- Sprint Review
- Sprint Retrospective

## 10. Increment vs MVP
- Increment: potentially releasable work each Sprint
- MVP: smallest set of features to learn from users

## 11. Velocity & Burndown
- Velocity: average points per Sprint
- Burndown: work remaining over time

## 12. Boards & Tools
- Kanban / Scrum Boards
- Jira, Asana, Trello, ClickUp
